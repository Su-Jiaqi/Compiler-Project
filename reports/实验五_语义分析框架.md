## 一、实验目的

本实验在词法分析和语法分析的基础上，实现一个简化的语义分析框架。语义分析的主要任务是维护符号表、进行基本语义检查，并构建抽象语法树表示，为后续中间代码生成提供信息。

## 二、实验内容

本实验完成以下功能：

1. 读取词法分析器输出的 Token 序列；
2. 对类 C 语言子集进行语法结构识别；
3. 构建简化 AST 输出；
4. 维护符号表，包括变量名、类型和偏移量；
5. 检查重复声明错误；
6. 检查变量使用前是否已经声明；
7. 为实验六生成四元式中间代码做准备。

## 三、支持的语言子集

本实验支持如下语句：

```c
int a;
float b;
int a = 3;
a = a + 1;
a += 1;
a++;
print a;
input a;
return a;
if (a < 10) { print a; } else { print b; }
while (a < 20) { a += 1; }
```

支持的表达式包括：

- 标识符；
- 整数和浮点常量；
- 加减乘除；
- 小括号表达式；
- 关系表达式；
- 简单函数调用表达式。

## 四、核心设计

### 1. 符号表

符号表使用 `map` 保存变量信息：

```cpp
std::map<std::string, Symbol> symtab;
```

其中 `Symbol` 包含：

```cpp
struct Symbol {
    std::string name;
    std::string type;
    int offset;
};
```

### 2. 语义检查

本实验实现了两个基础语义检查：

1. **重复声明检查**  
   如果变量已经在符号表中存在，再次声明时报告错误。

2. **未声明变量检查**  
   如果变量在表达式或赋值语句中被使用，但没有出现在符号表中，则报告错误。

### 3. AST 输出

程序会输出简化 AST 节点，例如：

```text
Program
Decl(int a)
Decl(int b)
Assign(a)
IfStmt
WhileStmt
ReturnStmt
```

该 AST 不是完整树形缩进结构，而是用于展示语义分析识别出的主要语法结构。

## 七、实验总结

本实验实现了一个简化的语义分析框架。通过符号表维护和基本错误检查，可以验证程序中的变量声明和使用是否合法。同时，AST 和符号表为后续中间代码生成提供了必要信息。虽然本实验没有实现完整类型系统和复杂作用域嵌套，但已经覆盖了语义分析的基本思想和主要流程。
