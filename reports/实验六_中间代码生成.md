# 编译器设计专题实验六：中间代码生成

## 一、实验目的

本实验在词法分析、语法分析和语义分析的基础上，实现中间代码生成。中间代码采用四元式形式表示，为后续目标代码生成、汇编代码生成或解释执行提供基础。

## 二、实验内容

本实验完成以下功能：

1. 对声明语句生成符号表信息；
2. 对赋值语句生成赋值四元式；
3. 对算术表达式生成临时变量和四元式；
4. 对关系表达式生成比较四元式；
5. 对 `if-else` 语句生成条件跳转和标签；
6. 对 `while` 语句生成循环标签和跳转；
7. 对 `print`、`input`、`return` 语句生成对应四元式。

## 三、四元式格式

本实验使用如下四元式格式：

```text
(op, arg1, arg2, result)
```

含义如下：

| 字段 | 含义 |
|---|---|
| op | 操作符 |
| arg1 | 第一个操作数 |
| arg2 | 第二个操作数 |
| result | 结果变量或跳转目标 |

例如：

```text
(+, a, b, t1)
(=, t1, , a)
(jz, t2, , L1)
(label, , , L1)
```

## 四、测试程序

输入源程序为：

```c
int a = 3;
int b = 5;
float c = 2.5;
a = a + b * 2;
if (a < 20) {
    print a;
} else {
    print b;
}
while (a < 25) {
    a += 1;
}
return a;
```

## 五、运行方式

```bash
make
./compiler ir input/test.src output
```

输出文件为：

```text
output/ir.txt
```

## 七、实验总结

本实验完成了简化类 C 语言子集的中间代码生成。四元式结构清晰，能够表达算术运算、赋值、条件跳转、循环、输入输出和返回语句。至此，实验一到实验六形成了一个完整的编译器前端和中间代码生成流程：DFA、词法分析、LR(0) 项目集、SLR(1) 分析表、语义分析和中间代码生成。
